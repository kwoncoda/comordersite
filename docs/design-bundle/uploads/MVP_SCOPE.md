## 1. MVP 정의

**MVP = "2026-05-20(수) 16:30 학교 축제 1차 운영에 *반드시* 작동해야 하는 최소 시스템"**

학교 축제 D-day 운영 중 누락 시 *부스 운영 자체 불가*한 기능만 P0. 운영 가능하지만 사용자 만족도 떨어지는 항목은 P1, 1차 운영 후 회고에서 결정할 항목은 P2.

### 1.1 MVP 성공 기준

1. **5/20 16:30에 사이트가 가동되고 메뉴 화면 진입 가능**
2. **사용자가 메뉴 선택 → 주문 → 도그태그 발급까지 무사**
3. **사용자 이체 후 → 본부 담당자가 통장 매칭 → "이체 확인" 클릭으로 상태 전이**
4. **조리 시작/완료/수령 토글이 운영진 화면에서 작동**
5. **사용자가 조리 현황판에서 본인 진행 상황 SSE로 확인**
6. **마감 시 진행 주문 0건 가드 + 정산 화면 + ZIP 다운로드**
7. **Docker 컨테이너가 죽어도 자동 부활 + 2시간 자동 ZIP 스냅샷이 남아있음** (2026-05-13 A 결정)
8. **5/20·5/21 양일 각각 관리자가 "장사 시작" 버튼 클릭 시 사용자 주문 가능** (G13). 정산 마감 시 자동 CLOSED 전이
8. **가격 위변조 시도가 서버에서 거부됨 (Pattern B 4 케이스 회귀)**

### 1.2 MVP가 *아닌* 것

- "완벽한 시스템" — 1차 운영 후 회고로 개선 ✅
- "모든 사용자 시나리오 커버" — P0 13개 화면 + P1 4-5개로 충분
- "Phase 2 기능까지 포함" — 의식적으로 deferred
