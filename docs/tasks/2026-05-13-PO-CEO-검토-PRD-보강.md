**6 검토 기준 답 요약:**

1. **진짜 문제인가** → 조건부 YES. 정산 자동화 + 데이터 보존이 핵심, 나머지는 보조
2. **사용자 동기** → YES (줄 우회 + SNS 공유). 운영 정책 (현장 주문 거부) 필요
3. **MVP 너무 큰가** → 코드 OK, 디자인 자산이 일정 위험. fallback 정책 필요
4. **반드시 빼야 할 것** → 없음. 축소 후보 7종 (A-G), 효과 큰 순 cherry-pick
5. **첫 출시 최우선 기능 1개** → 본부 매칭 (이체 확인) 흐름 — SPOF
6. **차별화 충분한가** → YES (학생 시장). 도그태그·WINNER WINNER·인스타 SNS 효과

### 2. PRD.md 보강 (5개 변경 적용)

`/plan-ceo-review`가 *반드시* PRD에 반영해야 한다고 식별한 항목만 적용. 단순 추천은 PO_REVIEW.md에만.

**Edit 1: §4.4 북극성 시나리오 (신규)**
- 위치: §4.3 학생회 운영 가치 다음
- 내용: "1개 흐름만 작동해야 한다면 = 메뉴 → 카트 → 폼 → 도그태그 → 본부 매칭 → 조리 → 수령". 보조 기능 명시 (도그태그 모션·WINNER WINNER·인기 랭킹·마스코트·카모·단축키·외부인 공유)
- 효과: 일정 압박 시·자산 미수령 시·D-day 사고 시 *어디부터 살릴지* 명시

**Edit 2: §7 KPI 재정의**
- 변경: "주문 처리 시간 30초 → 10초" (측정 불가) → "≤ 90초"
- 신규: "피크 폭주 처리량 ≥ 3 orders/min" (vs 직접 주문 ≤ 1 orders/min)
- 효과: 시스템 핵심 가치 = *폭주 분산*이 KPI로 명시됨

**Edit 3: §10.1 fallback 우선 정책 (신규)**
- 위치: 자산 표 다음
- 내용: 마스코트·도장·카모·메뉴 일러스트·약도·도그태그 6종 미수령 시 *기본 동작* 명시
- 효과: D-3 자산 미수령 시 자동 fallback 진입 — 일정 위험 완화

**Edit 4: §11.1 운영 리스크 격상 (SPOF 명시)**
- 변경: "본부 담당자 1명 병목"을 평범 항목 → 🔴 SPOF 첫 줄 격상
- 추가 완화책 4종: Buddy System / 백업 카뱅 단말 / HOLD 자동 강조 / 시스템 다운 fallback
- 신규 항목: 🟡 사용자 시스템 우회 (현장 직접 주문 거부 운영 정책)
- 신규 항목: 🟡 디자인 자산 일정 지연

**Edit 5: §13 변경 이력 갱신**
- 본 검토 결과 라인 추가
