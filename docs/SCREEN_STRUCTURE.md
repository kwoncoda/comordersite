## 3. 화면별 정보 위계

각 화면의 *1순위 (가장 큼·강조) → 2순위 → 3순위 (맥락)*. 사용자 시선이 닿는 순서.

### 3.1 C-1 메뉴 목록 (`/menu`)

```
┌─────────────────────────────────────┐
│ 헤더: 로고 + "오늘 저녁은 치킨이닭!"   │ 3순위 (브랜드)
├─────────────────────────────────────┤
│ 분류 탭: 전체·🔥인기·⭐추천·치킨...   │ 2순위 (탐색)
├─────────────────────────────────────┤
│ 🔥 실시간 인기 메뉴 TOP 3             │ 2순위 (사회적 증거)
│ 1위 뿌링클 37개 (정적 BEST, 2026-05-13 E)│
├─────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐              │
│ │ [사진]   │ │ [사진]  │             │ 1순위 (메뉴·가격)
│ │ 후라이드 │ │ 양념     │            │ ★ 가장 큼
│ │ 18,000원│ │ 19,000원│              │
│ │ [+ 카트] │ │ [+ 카트]│             │
│ │ ⭐ 추천  │ │ SOLD OUT│              │ 2순위 (배지)
│ └─────────┘ └─────────┘              │
│ ...                                   │
├─────────────────────────────────────┤
│ [Sticky 카트 바: "2개 21,000원 보기"] │ 1순위 (CTA, sticky)
└─────────────────────────────────────┘
```

**정보 위계:**
1. 메뉴 카드 (사진·이름·가격·CTA) — 가장 큼·중앙

3. 헤더 로고 — 한 번 인식 후 무시 가능

## 5. 네비게이션 패턴

### 5.1 사용자 네비게이션 — *최소화*

**원칙:** 사용자는 *주문 흐름의 직선*만 따라감. 메뉴 외 곁가지 X.

| 위치 | 표시 | 비고 |
|---|---|---|
| `/menu` | 좌상단 로고 (홈) | 클릭 시 새로고침 (다른 페이지 안 감) |
| `/menu` 외 | 좌상단 ← 뒤로 (브라우저 history) | 마스코트 X |
| 햄버거 메뉴 | **없음** | 메뉴 1개 직선 흐름이라 불필요 |
| 푸터 | 없음 | 모바일 화면 가치 X |

### 5.2 관리자 네비게이션 — *상단 가로 + 좌측 사이드*

```
┌────────────────────────────────────────────────────┐
│ [로고] 본부 / 메뉴 / 정산 / 쿠폰 / 시스템 / [logout] │ 가로 nav
├──────────┬─────────────────────────────────────────┤
│ 카드 5단계 │  현재 화면 본문                          │
│ 사이드 카운터│                                          │
└──────────┴─────────────────────────────────────────┘
```

- 5단계 카드 카운터는 *어느 화면에서든* 좌측 사이드에 작게 (ORDERED 1, TRANSFER_REPORTED 5 ⚠️ 등)
- 키보드 단축키 **4종 (Enter·Esc·Tab·?)** — 2026-05-13 D 결정으로 12종 → 4종 축소. chord(`g d`/`g t`/`g s`/`g m`)·숫자 점프·`h` 보류는 Phase 2 후보

### 5.3 모바일 vs PC 모드 분기 결정

| 화면 | 모바일 (≤768px) | PC (≥1024px) |
|---|---|---|
| `/menu` 등 사용자 | full-width 모바일 | (best-effort) 중앙 정렬 max-width 480px |
| `/admin/*` | (best-effort) 압축 — 본격 운영은 PC 권장 | full layout |
| 768-1024px | 둘 사이 어색 — 운영 가이드 "PC 또는 폰만" | — |

## 7. URL 컨벤션

| 패턴 | 의미 |
|---|---|
| `/{action}` | 사용자 정적 페이지 (`/menu`, `/cart`, `/checkout`, `/map`) |
| `/orders/:id/{action}` | 사용자 주문별 (`/orders/17/complete`, `.../status`) |
| `/admin/{action}` | 관리자 페이지 (`/admin/dashboard`) |
| `/admin/orders/:id` | 관리자 주문별 |
| `/api/*` | 사용자 JSON API (Alpine fetch) |
| `/admin/api/*` | 관리자 JSON API |

**쿼리 파라미터 (인증 fallback):**
- `?student_id=...&order_no=...` — 학생 인증 (쿠키 잃음 시)
- `?token=...&order_no=...` — 외부인 인증

> 보안: `student_id` URL 노출이 *완전 안전 X* — 단 학번을 모르면 타인 주문 조회 불가 + HTTPS 권장. 학교 와이파이라 HTTP인 경우는 운영 가이드에 명시.
